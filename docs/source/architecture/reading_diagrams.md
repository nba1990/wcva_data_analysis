# Reading & regenerating diagrams

This page explains how to read the architecture diagrams in this docs site and how to regenerate them from the codebase.

## What each diagram shows

- **System overview** (`system_overview`)
  High-level components: browser ↔ Streamlit server (`src/app.py`) ↔ data layer (`src/data_loader.py`, `src/eda.py`, `src/wave_context.py`) ↔ charts (`src/charts.py`, `src/sroi_charts/sroi_figures.py`) ↔ section pages (`src/section_pages/*`).

- **Execution flow** (`execution_flow`)
  Rerun model for `src/app.py`: page config → resolve dataset source → load and cache data → build sidebar + navigation → dispatch to a section page → run analysis helpers → build charts/tables → render UI → Streamlit reruns on interaction.

- **Streamlit pages** (`streamlit_pages`)
  Sidebar navigation items from `src/navigation.NavItem` mapped to the `render_*` functions in `src/section_pages/*`. Use this when adding or renaming pages.

- **Module/package dependencies** (`module_dependencies`)
  Package-level graph generated by `pyreverse` (`auto/packages.dot`), showing how top-level `src` subpackages depend on each other.

- **Class hierarchy** (`class_hierarchy`)
  Key classes and models from `src.config`, `src.navigation.NavItem`, and `src.wave_context`, rendered using Sphinx’s `inheritance_diagram` directive.

## Regenerating diagrams locally

From the project root, with the virtual environment activated:

```bash
source .venv/bin/activate
python scripts/generate_diagrams.py
cd docs
make html
```

This will:

- Run `pyreverse` to regenerate `docs/source/architecture/auto/classes.dot` and `docs/source/architecture/auto/packages.dot`.
- Rebuild the HTML docs so the updated Graphviz diagrams appear under the **Architecture** section.

## How Read the Docs uses this

On Read the Docs, the configuration in `.readthedocs.yaml`:

- Installs `requirements.txt` and `docs/requirements-docs.txt`.
- Runs `python scripts/generate_diagrams.py` in a pre-build job.
- Then runs Sphinx with `docs/source/conf.py`, which:
  - Enables `myst_parser` for Markdown.
  - Enables `sphinxcontrib.mermaid` so Mermaid diagrams render on RTD.
  - Enables `sphinx.ext.graphviz` and `sphinx.ext.inheritance_diagram` for code-derived diagrams.

As long as you keep `scripts/generate_diagrams.py` and `docs/requirements-docs.txt` in sync with the codebase, RTD will continue to build the diagrams automatically.
