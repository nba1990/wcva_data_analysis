# Learning Guides

These guides are for a Python/data-science reader using this repository as a
bridge into app development, deployment, testing, and public-repo hygiene.

They are intentionally grounded in this codebase rather than written as generic
"full-stack" theory notes.

## Recommended order

1. `01_repo_tour_for_python_data_scientists.md`
2. `02_private_data_secrets_and_demo_mode.md`
3. `03_testing_deployment_and_releases.md`
4. `04_from_notebook_to_production.md`

## Working style

- Keep the official guides curated and high-signal.
- Put temporary notes, chat exports, and rough drafts in `scratch/`, which is
  ignored from git.
- If a temporary note proves useful, distil it into one of these guides or into
  the README / architecture / ADRs instead of copying it verbatim.
